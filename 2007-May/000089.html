Summary: NT_STATUS_ACCOUNT_LOCKED_OUT error

Details: When I first access a share using smb4k, after entering a user name and password I receive the above error message in a dialogue box and I am unable to proceed.    Indeed,  for the next five minutes (roughly) I'm locked out of the NT domain completely.   When I try to access a Linux server  (which authenticates via the Active Directory kerberos server) using ssh or the samba server during this time period, here's what I see in the logs:

Aug  9 15:32:01 leibniz smbd[23816]:   domain_client_validate: unable to validate password for user aleahy in domain KNOX to Domain controller \\PUGSLY. Error was NT_STATUS_ACCOUNT_LOCKED_OUT.

Then after a few minutes, the error will miraculously clear up.  I've talked to the people who control the NT domain and they haven't seen anything in their logs.  On subsequent attempts to connect to the share, I don't have any problems.
