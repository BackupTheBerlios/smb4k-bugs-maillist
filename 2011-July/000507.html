Details: Hi,
First up, apologies for the inaccurate summary (there's a char limit)

Secondly: system info below[1]

The issue I'm experiencing is: 
When I first start smb4k, disable kwallet usage, then browse servers/shares via the Network Neighborhood, all the shares on multiple different types of servers (windows and linux) appear in the browse dialog without issue, however when I click on the share to mount it, it ALWAYS fails to mount with the error: 
"mount error(13): Permission denied
Refer to the mount.cifs(8) manual page"

I then have to right click, choose "Authentication" and enter my details. The shares will mount after this (except those I am not authorised for, in which case I get the same error above).

Version 0.9.x of smb4k would pop up an authentication dialog when clicking on the host - I'd like to see this functionality return -

Also, if possible it would be ideal if you could add a "Reauthenticate" button to the "permission denied" error dialog, so for users who are working on a kiosk type machine (of which we have several in boardrooms around the company) the error does not seem "fatal" (ie at the moment it does not suggest any further course of action eg trying to reauthenticate, how to authenticate etc). 

Regards, 
Bernie
