Summary: Should detect local charset

Details: I have these locales on both Samba server and client:
LANG=en_GB.UTF-8
LC_CTYPE=en_GB.UTF-8
LC_NUMERIC=fr_BE.UTF-8
LC_TIME=en_GB.UTF-8
LC_COLLATE=en_GB.UTF-8
LC_MONETARY=fr_BE.UTF-8
LC_MESSAGES=en_GB.UTF-8
LC_PAPER=fr_BE.UTF-8
LC_NAME=fr_BE.UTF-8
LC_ADDRESS=fr_BE.UTF-8
LC_TELEPHONE=fr_BE.UTF-8
LC_MEASUREMENT=fr_BE.UTF-8
LC_IDENTIFICATION=fr_BE.UTF-8
LC_ALL=

In smb.conf, I have:
   dos charset = 850
   unix charset = UTF-8

When I mount a share on the client with above UTF-8 locales, all special characters are not shown correctly. I have to configure smb4k to also use UTF-8 charset to make them show correctly.

smb4k should set the locale to use in CIFS based on the above locale variables, so this works out of the box.
