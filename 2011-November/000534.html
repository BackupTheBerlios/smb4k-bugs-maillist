Summary: Ping server before first mount attempt

Details: Hello,

I'm using Kubuntu 9.10 with network-manager and an encrypted WiFi connection. It would be great that, when opening, smb4k could "ping" the share servers before trying to initially remount the shares (or wait for a network-manager signal). It is needed because the WiFi connection takes some time to connect and sometimes smb4k starts before the connection is fully established.

We can imagine that if the shares are \\server1\share1 \\server1\share2 \\server2\share3, smb4k would have a thread that checks for server1 and server2 avaibility every 30s with smbclient and as soon as server1 is available, it remounts \\server1\share1 \\server1\share2 and as soon as server2 is available, it remounts \\server2\share3. 

It would be a benefit too for those who needs to establish VPN connection to their intranet.

What do you think ?

Barberousse.
