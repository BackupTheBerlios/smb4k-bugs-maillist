Summary: Improve visual feedback on succesful share mount

Details: The new tabbed interface hides the main visual feedback element of a succesful share mount. In v0.9.x you would get the slight icon change in the share browsing view, AND the share icon appearing in the right hand shares panel.

Now that tabs are being used to split out the dialogs, the visual feedback for a succesful mount is only the tiny icon change, as the Mounted  Shares tab is hidden in the background.

There are several possible solutions to this issue I would like to see (in order of preference):

* leave the layout as is, but if you change it, have it save the layout on closing, and restore on open so people can configure the layout any way they like (so I could do something like: http://i4.photobucket.com/albums/y131/clearyss/smb4k1.png and it would survive across restarts)
OR
* remove "Mounted Shares" from the tab view and keep it permanently visible as per screenshot above
OR
* upon succesful mount, switch to the mounted shares tab (I don't like this idea, as it will be a pita for anyone who needs to mount multiple shares at a time, however it would be easy to implement)

Hope this is makes sense -
