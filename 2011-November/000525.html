Summary: Let smb4k become a mainstream program

Details: If you like this tool to become standard in Ubuntu or similar, you' should split smb4k in a thin GUI and a 'deamon' process, which uses dbus for example to communicate. 
I see that the code is completely based on KDE/QT, so the daemon would initially depend on KDE, but if you use dbus as a communication mechanism, others could write a non-KDE deamon or a non-KDE GUI. 

You could also start with a very simple non-KDE deamon, and have the GUI do the most work, until you find a good balance between daemon on GUI.

If you don't do this yourselve, it's only a matter of time before a major distro sets up a competing program which does work like this, accross desktops.

Something to think about ?
