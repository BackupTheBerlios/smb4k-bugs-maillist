--- English version ---

Hello!

I assume you compiled the source yourself.

The error message you received occurrs when the libraries are not located 
under the prefix that is defined by the KDEDIR environment variable. Do you 
use a different directory for Smb4K? If so, I suggest that you either set the 
KDEDIRS environment variable in your ~/.bashrc file

export KDEDIRS=$KDEDIR:<prefix under which Smb4K is located>

or compile the source code with KDE's prefix directly

$ ./configure --prefix=`kde-config --prefix` ...

If you already use the default prefix, could you please tell me the 
distribution you are using?

The problem of the corrupted /etc/sudoers file is known and should be fixed 
since version 0.8.7. If you still experience this bug, could you please send 
me a (sanitized) copy of your /etc/sudoers so that I can identify and fix the 
problem.

Smb4K does not have laptop support at the moment, but it will be included in 
the near future.

Best regards
Alexander Reinholdt
