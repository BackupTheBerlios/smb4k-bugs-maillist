After some investigation I found out that this is not a bug of Smb4K itself, 
but a problem that occurs when using kdesu to read /etc/sudoers with root 
privileges. In the terminal output line breaks will be inserted where they do 
not belong. This is true for smb4k_cat as well as for cat. When using both 
programs without kdesu or e.g. with sudo everything is OK.

I don't know how to fix this external problem in Smb4K yet, but I'm working on 
it. A bug report to bugs.kde.org has been sent.
