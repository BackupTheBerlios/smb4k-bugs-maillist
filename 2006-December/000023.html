Summary: Per-mount setting: smbfs or cifs

Details: At the moment, there is only a global default whether to use SMBFS or CIFS for mounting. I'd like to have the possibility to overload this default per each mount point.

Reason:
There are heterogenous network with different Samba services, which support different protocols better.
Example:
- Samba 2.2.0
  Doesn't support CIFS at all, able to mount with SMBFS only.
- Windows 2003 Server
  Supports SMBFS without authentication only, don't know why. If I mount it with CIFS all types of authentication work.

Thanks,
René
