Summary: System/fstab mounts and User's mounts

Details: Hi!
The scenario:
A Multiuser/multiused Gentoo Box that use a remote LDAP directory for name services and authentication.

There's a "system" mount (CIFS) mounted at boot via fstab, mounting a remote samba share with unix extensions (-> and uid/gid coherence between client and server)
The mount command after the boot (no users logged in) shows:
//192.168.41.8/distfiles on /usr/portage/distfiles type cifs (rw,mand)

and is used by the gentoo's package manager.

The problem:
when a user login and launch smb4k for the first time, there are no problems, but if the user activate the option: "Remount recently used shares on program start", then logout, login again, and launch smb4k, it crash! In fact smb4k store the already fstab mounted share (/usr/portage/distfiles) as share to remount on program start, but it's already mounted and the application crash!

After the crash mount give:
//192.168.41.8/distfiles on /usr/portage/distfiles type cifs (rw,mand)
//192.168.41.8/distfiles on /home/plasma/Netmount/192.168.41.8/distfiles type cifs (rw,mand,nosuid,nodev,user=plasma)

So smb4k try to mount the "system" cifs share inside my personall Netmount folder! From my point of view smb4k should completely ignore the non-user cifs mounts!

I think smb4k should "understand" if a smb/cifs mount is a "system" mount or a user mount and avoid to try to manage it or remount it at startup when the option "Remount recently ecc ecc"!

Some info:
smb4k version: 0.8.3 (no debug symbols)
mount.cifs version: 1.10-3.0.24
kwallet is used for password storing

Please tell me if you need some stack trace, I can recomplie smb4k with debug symbols if useful!

Regards

Mark
