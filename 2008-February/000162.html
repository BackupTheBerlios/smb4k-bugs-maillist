I have smb4k in my session, so it starts once I log in. The passwords for my samba shares are stored in kdewallet. Every time there is a problem accessing kdewalled (e.g. wrong password or some other problem) smb4k falls back to asking me for passwords for all my shares. While being able to access my shares with a disfunctional/non responding/closed wallet is good, the option in the settings dialog is reset as well which is bad. Because the next time you log in the wallet problem should be fixed and smb4k should again try to use the wallet. If it is still broken, then it can fall back to asking for passwords. But the option should never change without user intervention.

I admit that my wallet problems may be due to misconfiguration on my side, and smb4k has to handle such problems (which it does). But resetting an option I specifically set without asking or at least telling me, is not very nice.
