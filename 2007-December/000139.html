cifs is different to smbfs:

mount.smbfs //server/share /mnt/point 
mount.cifs //server/share /mnt/point -o servern=netbiosname_of_server

There is no way to do this in smb4k, which means cifs mounting fails unless the netbios name is the same as the server name and is a TCP name. So if your server is called myserver and you have a share called fred then //myserver/fred will work, but if your server (myserver) has several virtual hosts set up in samba then you need to pass the virtual host name in the servern option.
